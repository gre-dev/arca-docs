Capturing work as a task, even an incomplete one, is always better than keeping it in your head or a side document. Create a task the moment something needs to get done, then fill in the details as you have them.

## Where to create a task

You can create a task from almost anywhere in Arca:

- Click the **+ New task** button in a list or board view.
- Press **N** while viewing a list or board (keyboard shortcut).
- From inside a task, use the **Subtasks** section to add a child task.

## What to set when creating a task

When you create a task, you can set the most important details right away:

- **Title** (required): keep it clear and action-oriented, e.g. "Fix login error on mobile" not "Login problem"
- **Status:** the workflow stage ("To do", "In progress", "Done")
- **Priority:** Urgent, High, Medium, or Low
- **Assignees:** who will own and work on it
- **Due date / start date:** when it should be completed or started
- **Labels:** tags for categorizing work ("bug", "design", "Q2")
- **List:** which list in your workspace the task belongs to

## Tips for capturing work efficiently

- **Capture first, refine later:** don't wait until you know all the details. Create the task with a title and assign it. You can always add a description, due date, and labels after.
- **Use clear, short titles:** teammates should be able to scan a list and immediately understand what each task is.
- **Assign immediately:** an unassigned task is an orphaned task. If you know who owns it, assign it on creation.
- **Set a due date:** tasks without dates disappear from calendars and timeline views. Even a rough estimate is better than nothing.
